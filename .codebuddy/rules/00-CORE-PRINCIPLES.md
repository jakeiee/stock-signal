# 核心原则（每次对话强制加载）

## 🔴 代码修改安全铁律
1. 修改前必读完整文件
2. 单次修改≤100行
3. 核心功能保护（analysis/*, report/*必须先测试）
4. Git安全网（重要修改前创建分支）

## 🟡 数据源管理铁律
1. 先调研后接入
2. 最多2个源（主+备）
3. 降级方案必须

## 🟢 记忆系统使用
1. 只记录unique learning
2. 错误必记录到ERRORS.md
3. 每周清理

## 🔵 模块开发规范
1. 实现标准接口（BaseModule）
2. 自注册机制
3. 步骤化
4. 独立可测

## ⚪ 自动调整机制
1. 规则自动提升
2. 数据源自动切换
3. 模块性能监控
