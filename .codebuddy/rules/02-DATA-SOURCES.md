# 数据源管理规范

## 核心原则

1. **先调研后接入**：接入任何外部数据源前，先调研免费、稳定的数据源
2. **最多2个**：同一数据最多保持 2 个数据源（稳定 + 备用）
3. **用户确认**：除用户提供准确的API/包外，都需调研后询问用户意见
4. **失效排查**：数据源失效时先排查原因，再决定是否接入新数据源

## 接入流程

```
1. 识别数据需求
2. 调研免费稳定的数据源（打印结果）
3. 分析对比（稳定性、完整性、响应速度）
4. 询问用户选择
5. 接入选定的数据源
```

## 降级策略

```
主数据源失效 → 备用数据源 → 排查原因 → 询问用户是否接入新数据源
```

## 禁止事项

- ❌ 不经调研直接接入新数据源
- ❌ 同一数据超过 2 个数据源
- ❌ 忽略数据源失效的根因排查
- ❌ 未获用户确认自行决定数据源

## 数据展示原则

1. **必须包含信号判断**：🟢买入/🟡中性/🔴卖出
2. **简要解读优先**：避免堆砌数字，给出明确判断
3. **历史百分位**：关键指标都要显示历史百分位（如：78.3%）
4. **时间戳明确**：所有数据都标注数据日期和更新时间

## 金融数据检索优先级

1. **第一优先**：`neodata-financial-search` skill（自然语言通用金融数据搜索）
2. **第二优先**：`westock-data` skill（腾讯自选股行情接口）
3. **第三优先**：公开信息检索（WebSearch）

## xalpha 使用规范

- 安装：`pip install xalpha`
- 指数代码格式：A股中证 ZZ930601，港股 HKHSTECH/HKHSIII
- 使用 `indexinfo(code='XX')` 创建对象
- 获取数据：`info.price`
- 技术指标：`info.ma()` / `info.kdj()` / `info.macd()` / `info.boll()` / `info.rsi()`
