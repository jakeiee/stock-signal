本文件记录项目开发中必须遵循的规则和约定，确保输出一致性和专业性。

## 一、信号体系规则

### 核心原则
- **规则明确**：所有指标都有明确的阈值判断标准
- **仓位直接**：资金信号直接输出具体的仓位建议
- **权重清晰**：综合信号使用明确的权重分配

### 指标阈值标准

| 指标 | 买入信号(🟢) | 中性信号(🟡) | 卖出信号(🔴) |
|------|-------------|-------------|-------------|
| **PMI** | ≥50（扩张） | 49-50（临界） | <50（收缩） |
| **CPI** | 0-3%（温和） | 3-5%（偏高） | >5%（高通胀） |
| **股息率** | >5%（高分红） | 3-5%（正常） | <3%（低分红） |
| **PE百分位** | <30%（低估） | 30-70%（合理） | >70%（高估） |
| **风险溢价** | >6%（高） | 3-6%（中等） | <3%（低） |

### 资金信号输出规则

资金信号必须直接输出明确的仓位建议，不做模糊表述：

| 市场成交额 | 信号 | 仓位建议 |
|-----------|------|---------|
| >30000 亿 | 🔴 | 0-10%（规避） |
| 20000-30000 亿 | 🟡 | 20%（谨慎） |
| 15000-20000 亿 | 🟡 | 40%（适度） |
| 5000-15000 亿 | 🟢 | 60%（积极） |
| ≤5000 亿 | 🟢 | 80%（激进） |

### 综合信号权重分配

```
综合得分 = 资金信号×30% + 基本面信号×40% + 政策信号×10% + 全球信号×20%
```

各维度评分范围：-2（极度悲观）到 +2（极度乐观）

## 二、报告输出规范

### 飞书展示原则
- **简洁优于详尽**：每个指标一句话概括
- **官方解读浓缩**：将官方解读放在指标后面，不单独开大块
- **视觉清晰**：使用表格和图标（🟢🟡🔴）增强可读性

### 终端输出格式
- **结构清晰**：使用分层缩进（├ / └ / │）
- **对齐整齐**：数值对齐到小数点后两位
- **信号明确**：每个指标都要有明确的🟢/🟡/🔴信号

### 数据展示原则
1. **必须包含信号判断**：🟢买入/🟡中性/🔴卖出
2. **简要解读优先**：避免堆砌数字，给出明确判断
3. **历史百分位**：关键指标都要显示历史百分位（如：78.3%）
4. **时间戳明确**：所有数据都标注数据日期和更新时间

## 三、项目模块规则

### dividend_monitor 模块

#### 动态仓位建议算法

```
综合得分 = 估值信号×50% + 市场温度×30% + KDJ技术×20%

仓位档位：
- 满仓：≥85%
- 重仓：70-85%
- 标配：50-70%
- 轻仓：30-50%
- 低配：10-30%
- 空仓：<10%
```

#### 估值信号评分（权重50%）

| 股息率百分位 | PE百分位 | 得分 |
|------------|---------|------|
| >70% | <30% | +2 |
| 50-70% | 30-50% | +1 |
| 30-50% | 50-70% | 0 |
| 10-30% | 70-90% | -1 |
| <10% | >90% | -2 |

#### KDJ技术信号评分（权重20%）

| J值范围 | 信号 | 得分 |
|--------|------|------|
| J < 10 | 🟢强烈超卖 | +2 |
| 10 ≤ J < 20 | 🟢超卖 | +1 |
| 20 ≤ J ≤ 80 | 🟡中性 | 0 |
| 80 < J ≤ 100 | 🔴超买 | -1 |
| J > 100 | 🔴极度超买 | -2 |

#### 数据源优先级

```
无风险利率：push2.eastmoney.com 实时
           └─ 备用：push2his.eastmoney.com 历史
           └─ 兜底：保底值 1.70%

估值数据：本地缓存（valuation_cache.json）
         └─ 缓存未命中 → 妙想 API
         └─ 配额用尽(status=113) → 标注"✗"

周线KDJ：中证官网OHLCV自算（优先）
        └─ 自算失败 → 妙想API（标注"⚠ 妙想降级"）

成交额：中证全指（000985）历史接口
```

## 四、技术实现规范

### Python代码规范
- 使用类型注解（Type Hints）
- 函数文档字符串遵循 Google 风格
- 异常处理要有明确的错误信息
- 关键操作要添加日志输出

### 数据缓存策略
- 估值数据缓存到 `valuation_cache.json`
- 缓存有效期：1天
- 缓存命中时直接使用，不消耗API配额
- 配额用尽时降级到缓存或自算

### API调用规范
- 使用重试机制（最多3次）
- 捕获配额用尽错误（status=113）
- 超时设置：10秒
- 失败时提供降级方案

### KDJ算法规范
- 使用标准参数：KDJ(9,3,3)
- 日线 → 周线重采样：`pandas resample('W-FRI')`
- RSV = (close - min_low_9w) / (max_high_9w - min_low_9w) × 100
- K = 2/3 × prev_K + 1/3 × RSV（初值50）
- D = 2/3 × prev_D + 1/3 × K（初值50）
- J = 3K - 2D

## 八、数据源接入规范

### 核心原则

1. **先调研后接入**：接入任何外部数据源前，先调研免费、稳定的数据源
2. **最多2个**：同一数据最多保持 2 个数据源（稳定 + 备用）
3. **用户确认**：除用户提供准确的API/包外，都需调研后询问用户意见
4. **失效排查**：数据源失效时先排查原因，再决定是否接入新数据源

### 接入流程

```
1. 识别数据需求
2. 调研免费稳定的数据源（打印结果）
3. 分析对比（稳定性、完整性、响应速度）
4. 询问用户选择
5. 接入选定的数据源
```

### 降级策略

```
主数据源失效 → 备用数据源 → 排查原因 → 询问用户是否接入新数据源
```

### 当前数据源清单（勿随意增加）

| 模块 | 数据项 | 主数据源 | 备用数据源 |
|:-----|:-------|:---------|:-----------|
| dividend_monitor | 指数估值 | Wind APP | 妙想API |
| dividend_monitor | 无风险利率 | 东财实时 | 东财历史/保底1.70% |
| dividend_monitor | KDJ指标 | 中证官网自算 | 妙想API |

### 禁止事项

- ❌ 不经调研直接接入新数据源
- ❌ 同一数据超过 2 个数据源
- ❌ 忽略数据源失效的根因排查
- ❌ 未获用户确认自行决定数据源
